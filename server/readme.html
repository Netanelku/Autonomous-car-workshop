FastAPI Car Control System

Overview

This project is a FastAPI application designed to manage and interact with a car system for object detection and retrieval. It provides endpoints for updating configuration, checking system health, and handling tasks related to object detection and alignment.

Installation

  1. Clone the repository: 
    git clone https://github.com/your-repository-url.git
cd your-repository-directory
  2. Set up a virtual environment: 
    python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate
  3. Install the required packages: 
    pip install -r requirements.txt
  4. Create a config.yaml file:

    Ensure you have a config.yaml file in the root directory with the following structure:

    car_address: 'http://192.168.1.100'
retry_attempts: 5
  5. Create a tasks.yaml file:

    Ensure you have a tasks.yaml file in the root directory with the following structure:

    tasks: {}

Configuration

  • config.yaml: Contains the configuration for the car system, such as the car's IP address and retry attempts.
  • tasks.yaml: Stores the status of various tasks.

Endpoints

UI Endpoints

  • GET /: Serves the README file content.
  • POST /car/updateRetryAttempts: Updates the retry attempts for the car system.
  • GET /car/currentAddress: Retrieves the current IP address of the car.
  • POST /car/updateAddress: Updates the IP address of the car.
  • GET /task/status/{task_id}: Retrieves the status of a specific task by its ID.

Car Configuration Endpoints

  • GET /health: Checks the health of the car server.

Task Management Endpoints

  • GET /camera/save_object: Saves an image of the specified object.

Main Operation Endpoints

  • GET /camera/locating_any_objects: Initiates the process to locate and align the target object and the starting point.

Running the Application

To start the FastAPI application, run the following command:

uvicorn main:app --host 127.0.0.1 --port 8080

Replace main with the name of your Python file if it is different.
